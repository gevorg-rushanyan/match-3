# Match-3 Game (Unity)

Небольшая match-3 игра, реализованная в рамках технического задания.  
Проект намеренно сделан максимально простым и читаемым, без использования сторонних библиотек и фреймворков.


apk: https://drive.google.com/file/d/1RNBHJLPJ145X72-d8o94KktxChgk8qRa/view?usp=sharing

---

## Технологии

- **Движок:** Unity 2022.3.62f2 (LTS)
- **Язык:** C#
- **Сцены:** 1 (`Main`)

Сторонние плагины **не используются**:
- DoTween
- UniTask
- Zenject

Вся логика реализована на стандартных возможностях Unity.

---

## Описание игрового процесса

- Классическая match-3 механика
- Игровое поле состоит из блоков разных типов
- Блоки перемещаются с помощью свайпов
- После каждого хода выполняется нормализация поля:
  - применяется гравитация
  - ищутся совпадения
  - совпавшие блоки уничтожаются
- Уровень считается пройденным, когда **все блоки уничтожены**

---

## Система сохранений

Прогресс игры полностью сохраняется между сессиями:

- текущий уровень
- полное состояние игрового поля

### Расположение файла сохранения
Application.persistentDataPath/game_save.json

- Формат сохранения: **JSON**
- Сохранение выполняется:
  - при сворачивании приложения
  - при закрытии приложения
  - после изменения состояния игрового поля

## Архитектура проекта

Проект построен с чётким разделением ответственности:

- **GameManager** — загружает конфигурации и инициализирует основные системы
- **GamePlayController** — управление игровым процессом и сценариями
- **BoardModel** — модель данных игрового поля
- **BoardSystem** — правила игры (move, gravity, matches)
- **BoardVisual** — визуальное отображение игрового поля
- **BoardController** — игровой поток (корутины, последовательность действий)
- **UIManager** — управление UI окнами
- **GameStats** — текущее состояние игры (уровень, события)
- **SaveSystem** — система сохранения и загрузки прогресса

UI и игровая логика взаимодействуют через события, без жёстких зависимостей.
